Week 8 Reading Notes

Transforms

  • The transform property allows you to translate, rotate, scale, and/or skew any element on the page.
  • We can manipulate an element's appearance using transform functions.
  • One or more functions separated by spaces that will be applied in the order they're provided.
  • Translate functions allow you to move elements up, down, left or right.
  • When you employ a translate function you're moving elements without impacting the flow of the document.
  • When you're only mocing an elemnt vertuically or horizontally, you can use the translatex or translatey functions.

    • Example: to move 45px to the right along the x axis you would write "transform: translateX(45px)

    Example: to move aong the y axis by 30px write "transform: translateY(-30px);

  • Transforms don't work on inline elements. To fix it add: display: inline-block;
  • The scale(x,y) function scales and elemnt bu the defined factors horizontally then vertically.
  • scale(1) would leave the element the same size, scale(2) would double its proportions, scale(0.5) would halve them.
  • You can use scaleX(x) or scaleY(y) functions.
  • A scaled element will grow outwards from or shrink inwards toward its center. To change the default behavior you can include the "transform-origin" property.
  • To declare multiple transformations provide a space-separated list .
  • If you scale inline-block elements then the trxt arounf it will fail to accommodate it with reflowing.
  • The rotate function rotates an element around the point of origin by a specified angle value.
  • By default the rotate function's point of origin is the element's center.
  • Positive degrees move clockwise and negative degrees more counter clockwise.
  • Declare the rotation before the translate so that it's applied first.
  • The skew(x,y) function specifies a skew along the x and y values.
  • Use the transform-origin property to control the origin from which the transforms are applied.
  • The order of transform functions matters, so if you rotate before translating then your translate direction will be on the rotated axis.
  • CSS transitions require less client-side processing than JavaScript so it ends up appearing smoother.
  • Transitions allow CSS properties to change over time.
  • CSS transitions are declared on an element.
  • Steps to creating a simple transition using only CSS:
    1. Declare the original state of the elemnt in the default style declaration.
    2. Declare the final state of your transitioned element such as a hover state.
    3. Include the transition functions in your default style declaration using the transition properties including: transition-property, transition-duration, transition-timing, transition-timing-function, and transition-delay.
    4. The transition-property property defines the properties that should be transitioned. All is the default.
    5. In order to use a transition you have to be able to find a valid midpoint.

    Example: A property being changed from solid to dashed doesn't have a valid midpoint and cannot be transitioned.

  • The -webkit- prefix needs to be included for all your transition properties to support older browsers.
  • The transition-duration property set how long the transition will take.
  • 200 milliseconds is considered the optimum time for a transition.
  • The transition-timing-function lets you control the pace of the transition.
  • The most common timing functions are ease, kinear, ease-in, ease-out or ease-in-out.
  • Using a cubic-bezier function helps set the transition more precisely.
  • There is a transition-delay property to set a transition to not start right away in case a page hasn't completely loaded.
  • The transition property allows for multiple transitions in one call.

Canvas, SVG, and Drag and Drop

  • We can draw shapes and lines, arcs and test, gradients and patterns with canvas.
  • Was first developed by Apple, then adopted by Mozilla and Opera, then standardized by the WHATWG.
  • Create a canvas element to start using.
  • A width and height must be set or it will be set by default to 300 width and 150 height.
  • Add a border to be able to see where it is on the page.
  • Use the getContext method and pass it a "2d" string.
  • Use strokeStyle for the stroke color and fillStyle for the fill color.
