Week 1 Notes, Questions, and Practice

Master Mobile UX Notes

  • Always design for mobile first
  • Steve Jobs said that a lot of people don't know what they want until they see it
  • "Find a way to fill their needs as simply as possible."
  • Things you can do to make a difference:
    1. Be there. If you're site or product is not on a mobile device then they will find an alternative.
    2. Be useful. Make it easy for them to find what they came for as quickly as possible or they will leave.
    3. Be quick. The loading must be quick and have good functionality. People get impatient.
  • Make it easy for people to do things with one hand.
  • Reduse image usage
  • When using images use SVG/Webfonts
  • Optimize and minify CSS and JS files
  • GZIP files to remove unnecessary data
  • Put primary controls at the bottom of the page/app for reachability
  • Make boxes on forms large enough that users can target it easily
  • Enable auto complete on forms to cut down on time it takes to fill out
  • Use a link tag for phone number so users can just tap on it to call
  • Use adverstising sparingly. Make it small.

Doing Stuff With Web Things Notes

  • A variable is like a box you put stuff in to help you organize and keep track of
  • To save information in local storage:
    function saveStory(){
    var storyName = document.getElementById("name_input").value
    var storyHTML = document.getElementById("story_editor").value
    localStorage.setItem(storyName, storyHTML)
    }
    The last line saves the information that was store in the "storyName" and "storyHTML" variables so that if the user leaves the page and comes back then the information will still be there.
    This is important to use when building apps to store the users information
