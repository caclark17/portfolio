Further Functions

  • Functions can have properties and methods for themselves.
  • Null needs to be provided as a function's first argument when it's not referring to an object as this.
  • Results can be saved in a cache so that when the argument is used multiple times it only has to comoute it once.
  • An IIFE is a function that is invoked as soon as it's defined by placing the whole function within parenthesis and placing parenthesis at the end of the function.

    • Example:

    (function(){ const temp='World'; console.log(`Hello ${temp}`); })();

  • An IIFE is used when the code only needs to be run once and won't be used again.
  • The recommended way to use strict mode is to put all of your code in an IIFE.
  • A recursive function invokes itself until a certain condition is met.
  • Callbacks are functions that are passed to other functions as arguments and then invoked inside the function they are passed to.
  • Event-driven Asynchronous Programming -- I don't quite understand this. I need to look up more info.
  • When a function is defined inside another function then the inner function will have access to any variables that are declared in the outer function's scope.
  • An asterisk symbol is placed after a function declaration to define a generator function.
  • Fundamental elements of functional programming are being able to pass functions as arguments, return them from other functions, and use anonymous functions and closures.
  • The use of pure functions are a key aspect of functional programming.
  • Using const to declare variables will help avoid destructive data transformations.
  • Pure functions only perform one task.
  • The hypotenuse function usees the square function to square numbers.
  • A callback can be used to find the sum of an array of numbers.
  • The sum, square, and mean functions can be used to build a variance function.
  • Higher-order functions are functions that accept another function as a argument or return another function.
  • A doubler multiplies a parameter by the argument that was provided to the multiplier.
  • The multiplier is a higher order function.
  • Currying applies the partial application of functions.
  • A function is considered curried when not all arguments have been supplied to the function.
  • Curried functions are higher order functions.
  • Currying can turn a single function into a series of functions.

AJAX

  • A technique that allows web pages to communicate asynchronously with a server.
  • Updates web pages without reloading.
  • Enables data to be sent and receives and things to be updated while the program continues to run.
  • Two parts to the interne-- clients and servers.
  • Ajax was coined by Jesse James Garrett in 2005 (JavaScript and XML).
  • The redirect method can be used to redirect to another URL.
  • JSON is the most common format for AJAX responses.
  • You can create your own response objects using a constructor function.
  • Requested objects include the properties: url, method, headers, mode, cache, credentials, and redirect.
  • A constructor function is used to create a new request object.
  • After a request object is assigned to a variable it can be used as the parameter of the fetch method.
  • has() can be used to checl if the headers object contains the header provided as an argument.
  • set() can be used to set a value of an already existing header.
  • append() adds a new header to the headers object.
  • delete() removes the header provided as an argument.
