Testing and Debugging

  • Errors are usually caused by system error, programmer error, or user error.
  • A program will continue running even after a warning is given.
  • JS Lint, JS Hint, and ES Lint are linting tools that are designed to test the quality of JS code.
  • Debugging is when a programmer finds bugs in their code so they can be fixed.
  • Using breakpoints can be helpful for finding bugs because it stops the execution of the code at certain points.
  • Using the alert() method will also stop the code from running at certain points until the x is clicked.
  • Modern browsers have tools that allow you to set breakpoints.
  • Using a debugger command will set a breakpoint in the code. I hadn't heard about this option until now.
  • Test-driven Development is when tests are written before writing code in order to use them to test the code once it is written. The tests will initially fail.
  • Jest can be used to test code. This can be downloaded in the terminal with "npm install -g jest".
