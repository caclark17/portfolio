Week 4 Reading Notes

Forms

  • document.forms will show the user all of the forms that are contained within the document.
  • form.elements will show the user which elements are used in the forms.
  • form.submit will submit the form automatically.
  • The value property can be used to get the text that's inside that field.
  • The placeholder attribute allows the user to see what they need to input into that box but disappears as the user begins to type.
  • Common types of form controls are input, select, textarea, and button.
  • The number of characters entered can be controlled by using the maxlength attribute.
  • When allowing the user to select more than one option from a drop-down list then the "multiple" acttribute is required.
  • Another type of button other than submit is a reset button.

    • Reset buttons are written: Vbutton type = 'reset'V Reset V/buttonV'

  • A submit button can be disabled if there are errors on the form by adding the "disabled" attribute.

Object-Oriented Programming in JavaScript

  • The 3 main concepts in OOP are encapsulation, polymorphism, and inheritance.
  • When the "instanceof" operator is used then it inherits the properties and methods that are defined in the function that it's referring to.
  • It's recommended to always use literals when creating arrays.
  • To see if an object has its own property you can use the hasOwnProperty() method.
  • Properties or methods that are inherited from it's prototype can be overwritten by assigning a new value to them.
  • Never use arrays or objects as a default value in prototype.
  • I can create an object using the properties of another variable. If I were to type const lois = Object.create(Human); then it would have the same properties as the word in parenthesis, which is Human. Human is also the prototype of lois.
  • init accepts initialization properties.
  • Creating objects from other objects creates a prototype chain.

      • Mixins

    • A mixin adds each property one at a time to the literal that has been created.
    • The mixin function allows a programmer to add a large number of properties to an object at one time.
    • A mixin function can also create a copy object. The object that is supposed to be copied goes in as a parameter.
  • A factory function can be used to return an object.
  • A mixin allows us to put multiple properties and methods in an object without creating an inheritance chain.
  • Keep inheritance chains short when creating classes.

Modular JavaScript

  • A module can be used by other modules and in other files and is a self-contained piece of code that provides functions and methods.
  • Coupling refers to a piece of code that can only run if another piece of code runs.
  • watch() method keeps an eye on the form and checks when it is submitted.
