Validating Forms

  • Client-side form validation helps to make sure that the information submitted matches the requirements.
  • All apps should perform security checks on both the client side and the server side.
  • Validation done in the browser is client-side validation.
  • Validation helps get the correct info, protest the user's data, and protect the business side from malicious users.
  • Validation attributes can be put on form elements.
      • Types of validation attributes
  • required
  • Minlength and maxlength
  • Min and max- specifies number of values
  • Type (number, email, etc)
  • Pattern
  • The browser will block the form from being sent if there is invalid information.
  • The constraint validation API has a set of methods and properties available.

Using the Fetch API

  • The Fetch API allows for accessing and manipulating parts of the the protocol like requests and responses.
  • Before this, XMLHttpRequest was used.
  • To set up a basic fetch request:

    • fetch('http://example.com/movies.json')

    .then((response) => response.json())

    .then ((data) => console.long(data));

  • Mulitple files can be accessed.
  • A text can be processed line by line.
  • A fetch promise will be rejected with a TypeError if it's not successful.
  • The request() accepts the same parameters as the fetch method.
  • The headers interface allows you to create your own headers with the Headers() constructor.
  • Request bodies can be set by passing body parameters.
